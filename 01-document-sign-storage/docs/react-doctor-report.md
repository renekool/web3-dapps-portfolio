## 2. Optimization Workflow

### Step 1: Solving the Critical Anti-pattern
We identified a `useEffect` that manually reset state when the connected address changed. This was replaced with the **Key Prop Pattern**. By adding `key={address}` to the component wrapper, React now handles the state reset automatically and correctly via unmounting/remounting.

### Step 2: Enhancing Accessibility (A11y)
Following React Doctor's suggestions, we audited interactive elements across the app:
- Added `role="button"` and `tabIndex={0}` to non-button interactive elements.
- Implemented `onKeyDown` handlers to support Enter and Space keys.
- Added descriptive `aria-label` attributes to icon-only buttons.

### Step 3: Architectural Refactoring
To improve maintainability and satisfy rule complexity limits:
- **`DocumentHistory.tsx`** was split into smaller, focused components: `DocumentTable.tsx` and `Pagination.tsx`.
- **State Management:** Multiple `useState` hooks were consolidated into `useReducer` to manage complex, related states (fetching, filtering, searching, and UI toggles) more predictably.

### Step 4: Infrastructure & SEO
- **Metadata Layouts:** Created server-side `page.tsx` wrappers for `Upload`, `Verify`, and `History` to export static SEO metadata, while keeping the interactive logic in client components.
- **Cleanup:** Removed 13 unused files and redundant utilities identified by the scanner to reduce bundle size and improve project clarity.
